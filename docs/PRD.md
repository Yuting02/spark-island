# 像素小镇 PixelTown · 产品方案（MVP）

> 原始需求见 [requirement.txt](../requirement.txt)。本文档是以 MVP 为目标的优化版产品方案。

## 1. 一句话定位

一个网页即开即玩的 2D 像素小镇：玩家每天进镇完成三件小事——**报亭读 AI 新闻、自习室读书记笔记、电台听播客**，把"每日信息摄入"游戏化。

## 2. MVP 取舍原则

| 保留（MVP 核心） | 砍掉 / 延后（V2+） |
|---|---|
| 单张小镇地图 + 键盘移动 + 三个可进入地点 | 多地图、NPC 对话、剧情 |
| 每日任务（3 项，按日期自动重置） | 成就系统、连续打卡、积分商城 |
| 昵称即玩（免注册，服务端发玩家 ID） | 账号体系、第三方登录 |
| 笔记保存云端、可回看 | 笔记富文本、导出、分享 |
| 管理后台：密码登录 + 上传新闻/epub/音频 | 多管理员、审核流、定时发布 |
| 电台播放器：播放/暂停/拖动进度/±15 秒 | 倍速、歌单、断点续听 |
| 桌面端键盘操作 | 移动端触屏适配 |

理由：MVP 要验证的是"像素小镇 + 每日任务"这个体验闭环是否成立，而不是游戏深度。所有延后项都不阻塞闭环。

## 3. 用户角色

- **玩家**：打开网址 → 输入昵称 → 进入小镇 → 完成每日任务。无需注册，玩家 ID 由服务端生成并存在浏览器本地。
- **管理员**：访问 `/admin` → 密码登录 → 上传/管理内容（新闻、书籍、电台节目）。

## 4. 核心玩法闭环

```
进入小镇 → 看到右上角"今日任务"面板（3 项）
   │
   ├─ 走到报亭，按 E → 报纸排版渲染当日 AI 新闻 → 任务①完成
   ├─ 走到自习室，按 E → 书架选书 → epub 阅读器 + 笔记面板
   │        └─ 保存一条笔记（云端） → 任务②完成
   └─ 走到电台，按 E → 选节目播放，累计收听 30 秒 → 任务③完成
   │
   └─ 三项全部完成 → 全屏庆祝提示；次日 0 点任务自动重置
```

## 5. 功能需求明细

### 5.1 小镇与移动
- 2D 俯视像素地图（程序绘制像素瓦片：草地、道路、树、水面、建筑）。
- WASD / 方向键移动，建筑与树有碰撞。
- 走到建筑门口出现"按 E 进入"提示，按 E 打开对应场景（DOM 浮层，非整页跳转）。

### 5.2 报亭（AI 新闻）
- 渲染管理员上传的当日新闻（Markdown），样式做成"报纸"：报头、日期、分栏正文。
- 当日无新闻时回退展示最近一期，并标注日期。
- 打开报纸即记任务①完成。

### 5.3 自习室（读书 + 笔记）
- 书架网格展示全部书籍（管理员上传的 .epub）。
- 点击书 → epub.js 在线分页阅读（上一页/下一页）。
- 阅读器旁是笔记面板：写笔记 → 保存到服务端（绑定玩家 ID + 书籍）。
- "我的笔记本"入口：按时间倒序回看自己的全部历史笔记。
- 保存一条笔记即记任务②完成。

### 5.4 电台（播客）
- 节目列表（管理员上传的音频：mp3 / m4a / mp4 / wav）。
- 自绘播放器：播放/暂停、可拖动进度条、快退/快进 15 秒、时间显示。
- 累计收听满 30 秒记任务③完成。

### 5.5 每日任务
- 服务端按"玩家 ID + 日期"存进度，跨设备一致；自然日切换自动重置。
- HUD 实时打勾，全部完成弹庆祝横幅。

### 5.6 管理后台（/admin）
- 密码登录（密码由环境变量 `ADMIN_PASSWORD` 配置，默认 `admin123`，部署时必须修改）。
- 新闻：标题 + 日期 + Markdown 正文，可删除。
- 书籍：标题 + 作者 + .epub 文件上传，可删除。
- 电台：标题 + 音频文件上传，可删除。
- 玩家端看到的都是前端渲染后的形态（报纸 / 书架+阅读器 / 播放器），不暴露原始文件管理界面。

## 6. 非功能要求（MVP 标准）

- 打开即玩：首屏 ≤ 2s（无构建框架、按需 CDN 加载阅读器库）。
- 数据持久化：JSON 文件存储（零外部依赖），数据量到千级玩家前够用；升级路径是换 SQLite/Postgres，存储层已隔离。
- 部署：代码托管 GitHub；运行需 Node 环境（本地 `npm start`，或 Render/Railway 一键部署）。
