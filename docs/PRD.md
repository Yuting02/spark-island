# 星火岛 Spark Island · 产品方案（v1.4）

> 原始需求与历次调整见 [requirement.txt](../requirement.txt)。本文档是以 MVP 为目标的优化版产品方案。
> v1.0 → v1.1 → v1.2 → v1.3 → v1.4 变更摘要见文末。

## 1. 一句话定位

一个网页即开即玩的动森风 2D 海岛社区：玩家每天上岛完成三件小事——**报亭读 AI 新闻、书屋读书记笔记、电台听播客**，赚取金币去**咖啡馆**犒劳自己，把"每日信息摄入"游戏化。

## 2. MVP 取舍原则

| 保留（MVP 核心） | 砍掉 / 延后（V2+） |
|---|---|
| 户外海岛 + 四个可进入的室内场景 | 更多地图、剧情任务线 |
| 每日任务（3 项，按日期自动重置）+ 金币奖励 | 成就系统、连续打卡、排行榜 |
| 昵称即玩（免注册）+ 邮箱绑定奖励 | 密码/OAuth 账号体系 |
| 场馆 NPC 对话 + 户外游走行人（随机台词） | NPC 剧情、好感度、任务派发 |
| 咖啡馆单人点单消费 | 多用户同屏社交、好友、聊天 |
| 笔记云端保存、可回看；阅读计时奖励 | 笔记富文本、导出、分享 |
| 管理后台：密码登录 + 上传新闻/epub/音频 | 多管理员、审核流、定时发布 |
| 桌面端键盘操作 | 移动端触屏适配 |

## 3. 用户角色与登录（v1.2）

- **账户玩家**：每次进入需在登录页（产品名只在此展示）用用户名+密码登录；金币、笔记、任务进度云端保留，下次登录继续。注册即送 10 金币见面礼；绑定邮箱一次性 +100 金币（MVP 只做格式与唯一性校验，不发验证邮件）。
- **游客**：一键进入，每次都是全新身份，进度不保存（界面有明确提示）。
- **管理员**：自行访问 `/admin`（玩家界面不出现任何后台入口）→ 密码登录 → 上传/管理内容。

## 4. 核心玩法闭环

```
上岛（户外海岛，WASD 移动，走到建筑门口自动进入室内）
   │
   ├─ 报亭室内：和老周聊天 → 走到报刊架按 E → 报纸排版读 AI 新闻 → 任务① +10🪙
   ├─ 书屋室内：走到书架挑一本书 → 走到自习桌坐下 → epub 阅读 + 记笔记
   │        ├─ 保存一条笔记 → 任务② +10🪙
   │        └─ 当日累计阅读满 30 分钟 → 额外 +10🪙（每日一次）
   ├─ 电台室内：走到控制台按 E → 播放器收听，累计 30 秒 → 任务③ +10🪙
   └─ 咖啡馆室内：吧台看菜单 → 点咖啡/奶茶（9–35🪙）消费金币
   │
   └─ 三项任务全部完成 → 庆祝动画；次日 0 点重置
```

## 5. 功能需求明细

### 5.1 海岛与移动（v1.2：伪3D + 相机 + 鼠标）
- 户外是**伪3D 图片大世界**（ui-building/island.png 放大 2 倍，3344×1882）：上方为远景、下方为近景，**相机跟随玩家滚动**，玩家可以一直走，场景延伸而非固定一屏。
- 四栋建筑（ui-building 立绘：报亭/书屋/咖啡馆/电台）**分散放置、互不紧挨**，按"y 越靠下越大"的纵深公式缩放（近大远小）；**玩家靠近建筑时建筑还会平滑放大约 16%**，远离恢复——伪3D 的临场感。
- 建筑/NPC/玩家按深度（y）排序绘制，人物可以被建筑遮挡；人物体型也随纵深缩放。
- 操作双模式：**键盘**（WASD/方向键移动、E 互动）+ **鼠标**（点地面走过去、点建筑走到门口自动进入、点 NPC 走近自动对话、点家具走近自动交互）；分辨率 1920×1080，按窗口等比缩放。
- 走进小屋门口自动进入室内，踩室内门垫自动回到岛上。室内沿用瓦片场景（书架/自习桌/吧台/控制台为交互点），同样支持鼠标点击。

### 5.2 NPC 与对话
- 每个场馆一位常驻 NPC（报亭老周、书屋阿书、电台 DJ 阿波、咖啡馆店长摩卡），户外两位游走行人（小柚、阿森）。
- 走近 NPC 按 E 对话：动森式对话框（名牌胶囊 + 打字机逐字 + E 推进），NPC 会转身面向玩家。
- 行人随机走动（简单 AI：走一格/原地转向，避开障碍与门口）。

### 5.3 报亭（AI 新闻）
- 室内布置：报刊架（交互点）、柜台、老周。
- 翻看报刊架 → 报纸排版渲染当日新闻（Markdown），当日无新闻回退最近一期并标注。
- 打开报纸停留 3 秒记任务①完成。

### 5.4 书屋（读书 + 笔记 + 阅读计时）
- 室内布置：两排书架、三张自习桌、阿书。
- **两段式体验**：书架前按 E 挑书（书架 UI，含"我的笔记本"回看入口）→ 提示去自习桌 → 桌前按 E 进入阅读。
- 阅读界面：epub.js 分页阅读 + 笔记面板（云端保存）+ 今日阅读计时（mm:ss / 30:00）。
- 保存一条笔记记任务②完成；当日累计阅读满 30 分钟奖励 10 金币（服务端 30 秒心跳累计，每日一次）。

### 5.5 电台（播客）
- 室内布置：DJ 控制台（交互点）、DJ 阿波。
- 播放器：播放/暂停、可拖动进度条、±15 秒；累计收听满 30 秒记任务③完成。

### 5.6 咖啡馆（金币消费 / 社交铺垫）
- 室内布置：吧台（交互点）、店长摩卡、咖啡桌椅。
- 菜单 6 款饮品：美式 9 / 拿铁 15 / 卡布奇诺 18 / 摩卡 22 / 珍珠奶茶 28 / 店长特调 35（金币）。
- 点单走服务端记账：余额不足拒单并提示；账单可在岛民护照查看。
- 定位：后续多用户社交（同屏、聊天、请客）的场地，MVP 先打地基。

### 5.7 金币经济（全部服务端判定与记账）
| 来源/去向 | 金额 | 限制 |
|---|---|---|
| 首次上岛见面礼 | +10 | 一次 |
| 完成一项每日任务 | +10 | 每任务每日一次（一天最多 +30） |
| 书屋阅读累计 30 分钟 | +10 | 每日一次 |
| 绑定邮箱 | +100 | 一次，邮箱全岛唯一 |
| 咖啡馆点单 | −9 ~ −35 | 余额不足拒单 |

- 每笔变动记入交易流水；"岛民护照"（点左上角玩家卡片）可查余额与最近账单。

### 5.8 每日任务与 HUD（v1.2 布局）
- 服务端按"玩家 ID + 日期"存进度；HUD 实时打勾；全部完成庆祝动画。
- **左上：今日任务**（默认展开，点标题收起）；**右上：玩家昵称 + 金币**（点击打开岛民护照）；**左下：地图导航小地图**（默认展开，点标题收起；显示四栋建筑位置与玩家实时红点，室内时高亮当前场馆）；底部居中：操作提示。
- 游戏内不出现产品名与管理后台入口；"星火岛 Spark Island"只出现在登录页。

### 5.9 管理后台（/admin，与 v1.0 一致）
- 密码登录（环境变量 `ADMIN_PASSWORD`）；新闻（Markdown）/书籍（.epub）/电台（音频）上传与删除。

## 6. 非功能要求

- 打开即玩：无构建框架，美术全部代码绘制（仓库零图片资产），阅读器库按需 CDN 加载。
- 数据：JSON 文件存储，存储层隔离，升级路径为 SQLite/Postgres。
- 部署：代码托管 GitHub；运行需 Node 进程（本地 / Render / Railway）。

## 附：v1.4 → v1.5 变更摘要

1. **建筑放大两倍以上**（620/640/430 场景单位，约占屏宽 37%），分置四角，重叠处靠 y 排序呈现前后关系；人物与建筑比例恢复正常。
2. **小猫五官严格对照 new/user.png**：白色下半脸与两颊、大圆黑眼带高光、粉鼻 ω 嘴、米色编织项圈配粉花（替代铃铛）、白色四肢带粉爪垫、橘尾深色环纹、头顶三道条纹。

## 附：v1.3 → v1.4 变更摘要

1. **素材**：全部改用 `ui-building/new/`（真透明 PNG，无需抠图）；删除 cut 产物与抠图脚本管线。
2. **固定场景**：取消"延伸场景/相机滚动"，整座岛固定一屏呈现（猫咪后院式定镜头）。实现为固定逻辑坐标系（1672×941）+ 渲染层按窗口 cover 缩放铺满——固定场景与屏幕自适应同时成立。
3. **建筑与人物加大**：建筑约占屏宽 1/5～1/4，猫咪约占屏高 14%，符合猫咪后院的物件比例。

## 附：v1.2 → v1.3 变更摘要

1. **屏幕自适应**：取消固定 1920×1080，画布跟随窗口尺寸实时变化（含室内场景缩放自适应）。
2. **猫咪角色**：主角与全部 NPC 换成猫咪（user.png 单帧立绘无法逐帧动画，按其配色代码重绘可动画橘猫：四方向头部朝向 + 两腿交替迈步 + 尾巴持续摆动；原图用于登录页与护照头像。若需直接使用原画动画，需提供 4 方向 × 2 帧精灵图）。
3. **猫咪后院 2D 风格**：移除伪3D（纵深缩放/靠近放大），建筑与角色固定尺寸，扁平构图；保留按 y 排序的前后遮挡。新增素材预处理脚本（npm run assets）抠除立绘的渐晕背景与"假透明"棋盘格。
4. **删除地图导航**小地图。
5. **进入建筑改提示制**：靠近门口出现"Ⓔ 进入 ×××（或点击）"气泡，按 E 或点击进入；不再自动传送、不再放大建筑。

## 附：v1.1 → v1.2 变更摘要

1. **操作**：新增鼠标点击（行走寻路 / 进建筑 / NPC 对话 / 家具交互 / 选书等 UI 本就支持点击），与键盘并存。
2. **界面净化**：玩家界面移除产品名与"管理后台"入口；管理员自行访问 `/admin`。
3. **伪3D**：使用 ui-building 立绘素材 + 纵深缩放 + 深度遮挡（架构评估见 ARCHITECTURE.md §9）。
4. **1920×1080**：固定逻辑分辨率按窗口等比缩放；产品名只在登录页；金币右上、任务左上（可收起）。
5. **登录体系**：每次进入须登录或游客；账户玩家进度云端保留，游客每次初始化。
6. **素材接入**：ui-building 文件夹（book/coffee/news/radio/login/island），建筑摆放近大远小。
7. **靠近建筑放大**：距离驱动的平滑缩放（仅视觉，碰撞不变）。
8. **延伸场景**：大世界 + 相机跟随，四栋建筑分散布局。
9. **地图导航**：左下角小地图，默认展开可收起。

## 附：v1.0 → v1.1 变更摘要

1. 改名：像素小镇 → **星火岛**；自习室 → **书屋**。
2. 视觉：像素风 → **动森风**（架构评估：核心架构不变，渲染层与引擎模块级改造，详见 ARCHITECTURE.md §8）。
3. 交互：进建筑不再直接弹功能窗 → **室内场景 + 家具交互点**（书架挑书 → 自习桌阅读的两段式体验）。
4. 新增**咖啡馆**场景与点单消费。
5. 新增**金币经济**（任务/阅读/邮箱/见面礼四种来源 + 消费记账）。
6. 新增 **NPC 系统**（四位场馆 NPC 对话 + 两位户外行人游走）。
