# 星火岛 Spark Island 🏝️🐱

一个网页即开即玩的猫咪后院风 2D 海岛社区：扮演一只橘猫岛民，每天上岛完成三件小事——**报亭读 AI 新闻 📰、书屋读书记笔记 📚、电台听播客 📻**，赚金币去**咖啡馆 ☕** 犒劳自己，把"每日信息摄入"游戏化。

- 产品方案：[docs/PRD.md](docs/PRD.md)
- 架构与文件说明：[docs/ARCHITECTURE.md](docs/ARCHITECTURE.md)

## 快速开始

```bash
npm install        # 安装依赖
npm run seed       # 生成种子内容（示例新闻 / 示例 epub / 示例音频）
npm start          # 启动，默认 http://localhost:3000
```

- 游戏入口：`http://localhost:3000`
- 管理后台：`http://localhost:3000/admin`（默认密码 `admin123`，生产环境务必通过环境变量修改）

## 玩法

1. **登录上岛**：注册账户（用户名+密码，送 10 金币见面礼）后金币、笔记、任务进度云端保留，下次登录继续；也可游客一键进入（每次都是新身份，进度不保存）。
2. **猫咪海岛（2D 固定场景，屏幕自适应）**：整座岛固定一屏呈现（猫咪后院式定镜头），任意窗口尺寸自动铺满。**鼠标点击**地面/建筑/角色即可行走与互动，或用 WASD / 方向键移动；走近建筑门口出现"进入"提示，按 **E** 或点击进入，室内踩门垫回到岛上。角色是会迈步、转头、甩尾巴的猫咪，岛上还有不同花色的猫咪 NPC：
   - **报亭**：和老周聊聊，翻报刊架读今日 AI 新闻 → 任务① `+10🪙`
   - **书屋**：书架挑一本书 → 自习桌坐下 epub 阅读、写云端笔记 → 任务② `+10🪙`；当日累计阅读 30 分钟再 `+10🪙`
   - **电台**：DJ 控制台点播节目（可拖动进度、±15 秒），听满 30 秒 → 任务③ `+10🪙`
   - **咖啡馆**：吧台点单消费金币（美式 9 ～ 店长特调 35），为后续社交玩法打地基
3. **HUD**：左上今日任务（可收起）、右上昵称+金币（点击打开**岛民护照**：绑邮箱一次性 `+100🪙`、查账单）。
4. 三项任务全完成有庆祝彩蛋；每天 0 点自动重置。
5. 美术资产：以 `ui-building/v1.6/island-new.png`（排版好建筑布局的整图）为视觉基准——整图作底图，每栋建筑用手工标定的遮挡多边形做 clip 裁剪重绘实现走动图层遮挡（零图像预处理，换图只需在 [world.js](web/js/game/world.js) 重标多边形顶点）；猫咪角色与室内为代码绘制（v1.6/user.png 立绘用于登录页与头像）。

## 管理后台

`/admin` 密码登录后可上传与删除（玩家端看到的都是渲染后的形态：报纸 / 书架与阅读器 / 播放器）：

| 内容 | 格式 |
|---|---|
| AI 新闻 | 标题 + 日期 + Markdown 正文 |
| 书籍 | 标题 + 作者 + `.epub` 文件 |
| 电台节目 | 标题 + 音频文件（mp3 / m4a / mp4 / wav / ogg） |

## 环境变量

| 变量 | 默认值 | 说明 |
|---|---|---|
| `PORT` | `3000` | 服务端口 |
| `ADMIN_PASSWORD` | `admin123` | 管理后台密码，**部署时必须修改** |

## 部署

代码托管于 GitHub。运行需要 Node ≥ 18 的进程环境（GitHub Pages 是纯静态托管，**无法**承载云端笔记/金币记账/管理上传，故不适用）。

本项目规模评估：单 Node 进程、JSON 文件存储、无外部数据库、静态资源 ~15MB、目标并发百人以内——**512MB 内存的最低档实例完全够用**。推荐顺序：

1. **Render**（首选）：免费档即可跑通，控制台全图形化。新建 Web Service 指向本仓库，Build：`npm install && npm run seed`，Start：`npm start`，环境变量配置 `ADMIN_PASSWORD`；给 `data/` 挂 1GB Persistent Disk（防止重新部署丢内容数据）。免费档闲置会休眠（首次访问冷启动 ~30s），上线给朋友玩够用；想免休眠可升 7$/月 Starter。
2. **Railway**：体验同级，按用量计费（每月 5$ 额度内基本免费），磁盘挂载同理。
3. **国内访问优先**：腾讯云轻量服务器 2C2G（~50¥/月），`git clone` 后 pm2 守护 + Nginx 反代 + 域名备案，访问速度最佳。
4. 不推荐 Vercel/Netlify（Serverless 函数无持久磁盘，文件存储模式不兼容）。

> 注意：`data/` 目录是运行时数据（数据库 + 上传媒体），不进 git；平台部署时给 `data/` 挂持久化磁盘，否则重新部署会丢内容。

## 技术栈

Node.js + Express + multer（后端，JSON 文件存储 + 服务端金币记账）；原生 Canvas 手写微引擎，动森风美术全部代码矢量绘制（仓库零图片资产）；epub.js + marked（CDN）。详见 [docs/ARCHITECTURE.md](docs/ARCHITECTURE.md)。
